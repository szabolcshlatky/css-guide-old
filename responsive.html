Conquering Responsive Layouts

Basics
Every HTML needs: <meta name="viewport" content="width=device-width, initial-scale=1.0" />
Use B__E--M methodology for classnames where its possible.
Use box-sizing: border-box; instead of the default content-box;.
Mobile first approach!
Widths & Heights
Avoid heights; for more background space use padding. Most of the case the default auto value is enough. If a hero needs to fill the screen then use 100svh
Avoid fixed unit widths, because children elements can overflow. With the basic auto value every page is fully responsive. 100% with margin causes overflow horizontally. CSS tries to keep information, but with overflow: hidden; we lose content.
width: 80%; means 80% of the parent element and with margin: 0 auto; it centers the block horizontally.
To make an image responsive, put it into a figure element and add 100% max-width to it.
Units
By browser default the root (html) element's font-size is 16px. 1rem = 16px (it checks the letter M). Best unit for font-size.
1em = the closest parent element's font-size (where is set up). Best unit for margin and padding.
min() & max() (Min() & Max() in SCSS) accept two or more values(even more min, max or math values with space between the operators). Use cases:
width: Min(70%, 500px); which is like width and max-width (the block won't be bigger).
width: Max(70%, 500px); which is like width and min-width (the block won't be smaller).
clamp() accepts three values as min() or max(). It can be used for: width: clamp(min-width, width, max-width); but the best usecase is for font-sizes.
width: max-content will grow the block's size to fit the content without break and overflows.
width: min-content will shrink the block's size based on the longest word. It is useful sometimes for grid.
width: fit-content won't overflow and will wrap the text.
@media type feature
@media print { CSS rules } changes the given styles when the page gets printed.
@media speech { CSS rules } changes the given styles when reading software is activated on the site for accessibility purposes.
@media screen (min-width: 900px) or @media (max-width: 900px) { CSS rules } changes the given styles depending on screen sizes.
Because @media is a boolean conditional statement it can look like this too: @media (min-width: 900px) and (max-width: 1000px)
Choose breakpoints by playing around with the DevTools and not the usual ones.
The most common breakpoints:
  • 600px
  • 900px
  • 1200px
  • 1800px
